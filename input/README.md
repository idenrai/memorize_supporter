# Memorize Supporter Input Data Guide

이 디렉토리(`input/`)에 JSON 파일을 추가하면 `npm run etl` 명령어를 통해 자동으로 데이터베이스에 덱(Deck)이 생성됩니다.
(하위 디렉토리까지 모두 재귀적으로 탐색합니다.)

## 디렉토리 구조 및 공유 정책

Github 등 외부에 코드를 공유할 때, 민감한 데이터와 공개용 데이터를 분리하기 위해 다음과 같은 구조를 사용합니다.

- **`public/`**: Github에 공유(커밋)되어도 안전한 공개 데이터 (예: 영단어, 생활 회화 등)
- **`private/`**: Github에 절대 올라가면 안 되는 민감한 비공개 데이터 (예: 상용 자격증 기출문제, 보안 지식 등)
- **`templates/`**: 덱 생성에 참고할 수 있는 템플릿 파일 모음

`.gitignore`에 의해 `private/` 폴더 내부의 모든 데이터는 무시되며, 오직 `public/` 및 `templates/` 만 버전 관리에 포함됩니다.

## 덱(Deck) 데이터 구조

각 JSON 파일은 반드시 하나의 최상위 **객체(Object)** 형태여야 하며, `cards` 배열 안에 개별 카드들을 정의합니다.
파일명과 관계없이 파일 내부의 `title`, `description` 속성이 화면에 표시됩니다.

### 🔄 데이터 업데이트 및 학습 기록 보존 (Stable ID)

`npm run etl` 명령어는 **기존 유저의 학습 진행도(망각 곡선 복습 주기)를 훼손하지 않고 안전하게 데이터를 업데이트(Upsert)** 합니다. 
이를 위해 시스템은 각 카드 객체에 선언된 `"id"`를 우선적으로 사용하지만, 만약 `id`가 명시되지 않았다면 카드의 **고유 내용(문제 텍스트, 단어 등)을 기반으로 MD5 해시 식별자(Stable ID)**를 자동 생성합니다.

- **학습 기록이 유지되는 경우:** 정답(`back`, `answers`), 해설(`explanation`), 선택지(`options`) 등을 수정하거나, 파일에 완전히 새로운 카드를 추가할 때.
- **학습 기록이 초기화(새 카드로 인식)되는 경우:** 카드의 핵심 텍스트(문제 내용인 `front`, `question`, `word`)를 수정할 경우, 시스템은 이를 전혀 다른 새로운 카드로 인식하여 기존 복습 주기가 초기화됩니다. 오타 수정 시 이 점에 유의하시거나, 처음부터 고유 식별자인 `"id"` 필드를 객체마다 직접 지정해 두는 것을 권장합니다.

```json
{
  "title": "AWS SAP C02 실전 모의고사",
  "description": "AWS Solutions Architect Professional 자격증 취득을 위한 문제 모음입니다.",
  "type": "practice_quiz",
  "cards": [
    // 아래 카드 타입 중 하나를 선택하여 배열로 나열
  ]
}
```

(참고: 하위 호환성을 위해 구버전 형태인 `[ {...}, {...} ]` JSON 배열만 작성해도 시스템이 자동으로 인식합니다. 단, 이 경우 덱 이름은 파일명으로 고정됩니다.)

## 지원하는 카드 타입 (Card Types)

`cards` 배열 내의 각 객체는 덱에 선언된 `type`을 기본으로 따르지만, 객체 내부에 `"type": "..."`을 명시하여 개별 카드의 형태를 덮어쓸 수도 있습니다.

### 1. 일반 암기 카드 (Flashcard)
간단한 질문과 답변을 적는 일반적인 플래시카드 형식입니다. (기존 tip)

```json
{
  "type": "flashcard",
  "category": "분류(선택)",
  "front": "카드 앞면 내용 (질문)",
  "back": "카드 뒷면 내용 (정답)"
}
```

### 2. 영단어장 (Vocabulary)
단어 암기를 위한 전용 레이아웃이 제공되는 형식입니다.

```json
{
  "type": "vocabulary",
  "word": "영단어",
  "meaning": "뜻",
  "example": "예문 (선택)"
}
```

### 3. 실전 모의고사 (Practice Quiz)
N지 선다형 문제와 복수 정답, 해설을 지원하는 객관식 형식입니다. (기존 multiple_choice)

```json
{
  "type": "practice_quiz",
  "question": "문제 내용",
  "options": ["선택지 1", "선택지 2", "선택지 3", "선택지 4", "선택지 5"],
  "answers": [0, 2], // 정답인 선택지의 인덱스 번호 배열 (0부터 시작)
  "explanation": "해설 내용 (선택)"
}
```
