# System Architecture

이 문서는 프로젝트의 시스템 아키텍처를 정의하기 위한 템플릿입니다.

프로젝트 시작 시, AI 에이전트에게 `/plan` 명령을 내리고 이 문서를 현재 프로젝트 스택과 요구사항에 맞게 갱신하도록 지시하세요.

## 1. 개요 (Overview)
- 시스템의 주요 목적
- 핵심 컴포넌트

## 2. 프론트엔드 (Frontend)
- 프레임워크 및 라우팅 방식
- 상태 관리 전략
- 주요 디자인 패턴

## 3. 백엔드 (Backend)
- 웹 프레임워크 및 API 라우팅
- 데이터베이스 및 ORM 연동 방식
- 인증 및 권한 부여

## 4. 인프라 및 배포 (Infrastructure & Deployment)
- 클라우드 제공자
- CI/CD 파이프라인
- 환경 변수 및 설정 관리
